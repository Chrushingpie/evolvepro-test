# Contribution Guidelines

Thank you for your interest in contributing to **EvolvePro**! Please follow these guidelines to help us maintain a clean, consistent, and high‑quality codebase.

## 1. Fork & Branch
- Fork the repository and clone your fork locally.
- Create a new branch for your feature or bug fix. Use a descriptive name, e.g., `feature/login-form` or `bugfix/fix-login-error`.

## 2. Code Style
- Follow the existing code style and formatting conventions used in the project.
- Run the linter (if available) before committing.

## 3. Commit Messages
- Write clear, concise commit messages.
- Use the imperative mood: "Add login form" instead of "Added login form".

## 4. Pull Request
- Submit a pull request (PR) from your branch to the `main` branch.
- Provide a descriptive title and a detailed description of the changes.
- Reference any related issues (e.g., `#42`).

## 5. Tests
- Ensure all tests pass before merging.
- Add new tests if you introduce new functionality.

## 6. Documentation & Screenshots
- Update documentation or add screenshots if your changes affect the UI or user experience.

Happy coding! 🚀
