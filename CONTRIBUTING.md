# Contribution Guidelines

Thank you for your interest in contributing to this project! Please follow these guidelines to help us maintain a consistent and high-quality codebase.

## 1. Fork & Branch
- Fork the repository.
- Create a new branch for your feature or bug fix: `git checkout -b feature/your-feature-name`.

## 2. Code Style
- Follow the existing code style and formatting conventions.
- Run the linter and tests before committing.

## 3. Commit Messages
- Write clear, concise commit messages.
- Use the imperative mood: "Add feature X".

## 4. Pull Request
- Submit a pull request with a descriptive title and description.
- Include any relevant documentation or screenshots if applicable.

## 5. Tests
- Ensure all tests pass before merging.
- Add tests for new features or bug fixes.

## 6. Documentation
- Update documentation if your changes affect usage.

Happy coding!
